style conventions for this project, feel free to make things fit this

modules: lowercaseletters
classes: UpperCamelCase
methods: lowerCamelCase
variables: lowercase_separated_by_underscores
constants: UPPERCASE_SEPARATED_BY_UNDERSCORES

strings: use single quotes (') for things acting as identifiers (example: dictionaries, file names)
use tripe double quotes for docstrings (""")
use double quotes otherwise (")

definitions: place a blank line (same indentation as one above) after function definitions
additional blank lines can be places for readability / structure
